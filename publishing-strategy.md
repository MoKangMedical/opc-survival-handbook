## 三、第二优先级：亚马逊KDP（被动收入）

参照文章流程，适配中文书：

### KDP上架流程
| 步骤 | 操作 | 工具/耗时 |
|------|------|-----------|
| ① 注册KDP账号 | kdp.amazon.com → 用邮箱注册 | 10分钟 |
| ② 收款绑定 | Payoneer / Wise → 美国虚拟银行 | 30分钟 |
| ③ 排版 | Canva KDP模板（6×9英寸） | 2-3小时 |
| ④ 封面 | Canva / Midjourney生成 | 1小时 |
| ⑤ 元数据 | 标题/描述/关键词（用AI优化） | 30分钟 |
| ⑥ 定价 | 成本$3-4 → 定价$9.99-$14.99 | 5分钟 |
| ⑦ 上架 | 提交审核 → 24-72小时上线 | — |

### Codex插件自动化发布SOP

手工发布最大的瓶颈不是写作，而是重复填写：分类、关键词、描述、作者信息、版权声明、纸张规格、封面和内页上传、预览检查。单本书还能忍，多本书批量发布时会迅速变成低价值体力活。

推荐把流程拆成两段：

1. **内容生成端**：使用已开源的AI写书Skill生成大纲、章节正文、书名、副标题、简介、关键词和分类候选。
2. **发布执行端**：使用Chrome浏览器中的Codex插件自动操作KDP后台，把元数据和文件逐项填入。

#### 操作步骤
1. 在Chrome浏览器安装并启用Codex插件。
2. 登录 Amazon KDP 后台，进入新建书籍页面。
3. 打开Chat界面，输入发布指令。
4. Codex插件自动填写书名、描述、分类、关键词、作者信息，上传内页PDF和封面文件。
5. 人工确认高风险项：版权声明、定价、区域授权、预览器检查、最终提交。

#### 推荐指令
```
Chrome，请帮我把当前我写完的书籍发布到亚马逊KDP。
请读取项目中的KDP PDF、封面文件和书籍元数据，自动填写分类、描述、关键词和上传文件。
定价、版权声明和最终提交前必须停下来让我确认。
```

#### 自动化收益
- 单本书发布从“手工逐项填写”变成“自动填写+人工确认”。
- 支持批量发布同系列书籍，尤其适合KDP低内容书、双语版、修订版和模板化选题。
- 保留人工审核关键节点，避免定价、版权、地区授权等高风险信息被误提交。

### KDP注意事项（中文书特有）
- 亚马逊KDP支持中文内容，但主要买家在海外华人市场
- 可出中英双语版，扩大受众
- 关键词用英文（如 "one person business"，"AI entrepreneur guide"）
- 描述用双语，增加搜索曝光

### 收入估算
```
定价 $12.99 × 月销 50本 = $649.50/月 收入
印刷成本 ~$3.5/本，版税 = $9.49/本
月利润: 50 × $9.49 = $474.50
年化: ~$5,700（放一年，被动收入）
```

## 四、第三优先级：国内出版

### 路线A：自助出版
| 平台 | 特点 | 适合 |
|------|------|------|
| 知乎盐选 | 专栏/电子书，平台流量大 | 快速验证 |
| 得到/樊登 | 需要编辑对接，品控严格 | 品牌背书 |
| 豆瓣阅读 | 自助上架电子书 | 口碑传播 |

### 路线B：传统出版
- **电子工业出版社 / 机械工业出版社** → 科技/创业类强项
- 投稿方式：完整大纲+样章 → 编辑邮箱
- 周期：6-12个月
- 版税：通常 6-10%

### 路线C：自媒体连载
- 微信公众号付费阅读
- 即刻/小红书分段发布 → 引流到完整版
- B站视频版 → 扩大传播
